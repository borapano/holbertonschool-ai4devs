# Reflection on AI-Assisted Specification Writing

## Introduction
In this project, AI was used to assist in developing a structured specification for the CarpoolConnect platform, a corporate carpooling system designed to help employees share rides, reduce commuting costs, and decrease carbon emissions. The assignment involved producing multiple specification artifacts, including a product idea description, user stories, technical specifications, API endpoints, and a system architecture diagram. AI tools were used to generate the initial drafts of these artifacts, while human review ensured that the outputs met the assignment requirements. The process highlighted both the strengths and limitations of using AI for technical documentation and specification writing.

## AI Strengths
AI performed particularly well in generating structured and repetitive content. Tasks such as writing user stories, listing system components, defining data models, and generating API endpoints were completed quickly and followed the provided formats accurately. When prompts included a clear example or template, AI produced results that closely matched the expected structure. For instance, the prompt for user stories included a defined format with roles, goals, acceptance criteria, and priority tags. Because the structure was explicit, AI was able to generate multiple stories that were logically connected to the project concept.

Another strength was AI’s ability to expand a high-level idea into multiple specification elements. From a single product concept, it produced matching features, services, and architectural components. This helped transform the initial idea into a more complete system specification. AI also worked well when refining language or improving existing text. When comparing original and refined versions of requirements, AI produced clearer and more precise descriptions, which improved readability.

## AI Weaknesses
Although AI handled structured prompts effectively, it struggled with prompts that required strict constraints or deeper reasoning. For example, prompts requiring an exact word count or multiple conceptual sections sometimes produced incomplete results. In several cases the generated text either missed certain required topics or did not satisfy the expected word range.

Another weakness appeared when prompts were less structured. Without clear formatting guidance, AI occasionally produced vague or generic descriptions. Some early user stories lacked specific acceptance criteria or measurable conditions. These issues made the requirements less useful until they were manually refined.

In addition, AI occasionally introduced inconsistencies in terminology. For example, a service might be labeled as “Matching Service” in one section and “Commute Matching Service” in another. While this difference was small, consistent terminology is important in technical documentation and required manual correction.

## Human Role
Human involvement was essential for ensuring that the generated specifications met the assignment criteria. The primary role of the human reviewer was validating the AI outputs and adjusting them to match the required structure and constraints. This included verifying the number of user types, ensuring that each section followed the required format, and confirming that the reflection contained all necessary discussion points.

Humans also played an important role in improving clarity and specificity. When AI produced generic descriptions, manual editing helped add clearer system behavior and more precise acceptance conditions. Another key task was maintaining consistency across the document by standardizing component names and terminology.

Finally, humans were responsible for evaluating whether the generated specifications logically represented the intended system. AI can suggest plausible components and features, but human judgment is needed to ensure they align with the actual product concept.

## Lessons Learned
This project demonstrated that AI is a powerful tool for accelerating the early stages of specification writing. It is particularly effective when tasks involve structured formats, repetitive patterns, or template-based documentation. However, AI performs best when prompts are carefully designed with explicit formatting instructions and examples.

The experience also showed that human refinement remains essential. AI-generated drafts should be treated as a starting point rather than a final product. Careful review is necessary to ensure accuracy, clarity, and consistency.

For future projects, specification writing could benefit from improved prompt libraries that define reusable templates for user stories, technical components, and architecture descriptions. These templates would guide AI more effectively and reduce the need for extensive revisions. Overall, the combination of AI-generated drafts and human refinement provides an efficient workflow for producing high-quality technical specifications.
