## Code Quality

- **Refactoring**
  - **Goal**: Improve readability and performance  
  - **Input**: Source function in [LANGUAGE]  
  - **Output**: Optimized code + explanation 

- **Duplicate Code Detection**
  - **Goal**: Identify and remove repeated code
  - **Input**: Multiple code files or functions
  - **Output**: List of duplicates + suggestion for reusable function

- **Best Practices Review**
  - **Goal**: Ensure code follows programming best practices
  - **Input**: Source code
  - **Output**: List of improvements and corrected code examples


  ## Debugging

- **Error Identification**
  - **Goal**: Find the cause of an error in the program
  - **Input**: Code snippet + error message
  - **Output**: Explanation of the error and the problematic line

- **Bug Fixing**
  - **Goal**: Correct bugs in the code
  - **Input**: Buggy code
  - **Output**: Corrected version of the code + short explanation

- **Logic Error Detection**
  - **Goal**: Identify mistakes in program logic
  - **Input**: Code that produces incorrect results
  - **Output**: Explanation of the logic problem + suggested fix

  
  ## Documentation

- **Function Documentation**
  - **Goal**: Generate clear documentation for a function
  - **Input**: Source code of a function
  - **Output**: Description of the function, parameters, and return value

- **Code Comment Generation**
  - **Goal**: Add helpful comments to improve code understanding
  - **Input**: Code snippet
  - **Output**: Code with explanatory comments

- **README Creation**
  - **Goal**: Create a basic README file for a project
  - **Input**: Project description and main files
  - **Output**: Structured README with overview, setup, and usage


  ## Testing

- **Unit Test Generation**
  - **Goal**: Create unit tests for a function
  - **Input**: Source code of a function
  - **Output**: Unit test cases for the function

- **Test Case Creation**
  - **Goal**: Generate different test cases to validate program behavior
  - **Input**: Program description or code
  - **Output**: List of test cases with expected results

- **Edge Case Detection**
  - **Goal**: Identify unusual or extreme inputs that should be tested
  - **Input**: Code or function description
  - **Output**: List of edge cases to include in tests
